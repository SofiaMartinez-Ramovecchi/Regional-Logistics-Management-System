# ADR 008 — Implementación del TDA Grafo mediante Matriz de Adyacencia

## Estado

Aceptado.

---

# Contexto

El sistema requiere modelar la red de sucursales y las rutas entre ellas para dar soporte a los servicios de expedición, recolección y cálculo de rutas óptimas (`ServicioExpedicion`, `ServicioRecoleccion`, `ServicioRutas`).

Se necesita una estructura que permita:

- Representar nodos (sucursales) y aristas ponderadas (rutas con distancia en km).
- Calcular caminos mínimos entre dos puntos de la red.
- Recorrer la red por niveles de cercanía para optimizar la recolección.
- Determinar la sucursal más cercana disponible para una expedición.

---

# Decisión

Se decidió implementar el TDA Grafo utilizando una **matriz de adyacencia ponderada** (`double[][]`), donde cada celda almacena la distancia en kilómetros entre dos sucursales. El valor `0` indica ausencia de conexión directa.

El grafo se define como **no dirigido**, dado que las rutas entre sucursales son transitables en ambas direcciones.

La clase implementadora es:

```text
GrafoMatrizAdyacencia
```

La cual implementa la interfaz:

```text
IGrafo
```

Métodos incluidos:

```text
insertarVertice(sucursal)
eliminarVertice(sucursal)
insertarArista(origen, destino, distancia)
eliminarArista(origen, destino)
existeVertice(sucursal)
existeArista(origen, destino)
dijkstra(origen)
caminoMinimo(origen, destino)
calcularRecorrido(inicio)
conectarPasillos(a, b)
verProximo(origen)
```

---

# Consecuencias

## Positivas

- Acceso en tiempo O(1) para consultar si existe una arista entre dos sucursales.
- Implementación de Dijkstra directa sobre la matriz sin estructuras auxiliares externas.
- Recorrido BFS por niveles apto para optimizar el orden de recolección en el almacén.
- Fácil integración con `ServicioRutas`, `ServicioRecoleccion` y `ServicioExpedicion`.

## Negativas

- Uso de memoria O(n²), ineficiente si la red crece y la mayoría de sucursales no están conectadas entre sí.
- La capacidad máxima de nodos debe definirse al momento de construcción.
- Dijkstra implementado con búsqueda lineal del mínimo: complejidad O(n²) en lugar de O((n + m) log n) con heap.
