# ADR 000 — Modelado de Lotes y Trazabilidad de Movimientos

## Estado

Aceptado.

---

# Contexto

La consigna del proyecto establece que:

> "Debe existir un registro de los últimos movimientos de mercadería que permita revertir una acción errónea en el inventario."

Durante el análisis inicial se evaluó realizar la trazabilidad directamente sobre los productos almacenados.

Sin embargo, se identificó que un mismo producto puede existir en múltiples partidas de mercadería con distintos ingresos, ubicaciones y recorridos dentro del centro logístico.

Por este motivo surgió la necesidad de distinguir entre el concepto de producto y el concepto de lote.

---

# Decisión

Se decidió modelar los lotes como entidades independientes de los productos.

La relación definida es:

```text
Producto 1 ---- N Lotes
```

Cada lote representa una partida específica de mercadería y posee identidad propia dentro del sistema.

Los movimientos logísticos se registrarán sobre lotes y no directamente sobre productos.

---

# Modelo Conceptual

```text
Producto
    |
    | 1:N
    |
Lote
    |
    | 1:N
    |
Movimiento
```

---

# Registro de Movimientos

Cada lote mantendrá un historial de movimientos realizados dentro del sistema.

Un movimiento podrá representar acciones como:

* ingreso al depósito,
* traslado entre sectores,
* preparación de pedidos,
* expedición,
* despacho.

Ejemplo:

```text
Lote L001

Recepción
↓
Depósito Central
↓
Pasillo 5
↓
Pedido 45
↓
Expedición
↓
Camión 3
```

---

# Reversión de Operaciones

Para cumplir con el requerimiento de reversión de acciones erróneas se utilizará una estructura tipo Stack (DynamicStack).

Cada nuevo movimiento registrado será almacenado en la pila.

```text
push(movimiento)
```

Ante un error operativo el sistema podrá recuperar el último movimiento realizado:

```text
pop()
```

y ejecutar la operación inversa correspondiente.

Esta estrategia sigue el principio LIFO (Last In, First Out), adecuado para implementar funcionalidades de deshacer operaciones.

---

# Consecuencias

## Positivas

* Permite trazabilidad completa de mercadería.
* Diferencia claramente productos de partidas físicas reales.
* Facilita auditoría de movimientos.
* Permite implementar reversión de operaciones utilizando pilas.
* Se alinea directamente con los requerimientos de la consigna.

---

## Negativas

* Incrementa la cantidad de entidades del dominio.
* Requiere mantener relaciones entre productos, lotes y movimientos.
* Aumenta la complejidad del modelo respecto a una gestión basada únicamente en productos.

---

# Justificación

Se considera que la trazabilidad de lotes representa de forma más precisa el flujo real de mercadería dentro de un centro logístico y permite satisfacer el requisito de registro y reversión de movimientos solicitado por la consigna.
