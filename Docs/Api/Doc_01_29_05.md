# Documentación de API — Versión 1.0

## Centro Logístico de Distribución Avanzada

# Introducción

La API del sistema se encuentra desarrollada utilizando Spring Boot y es consumida mediante el prefijo:

```text id="r8q4u2"
/api
```

configurado a través de Nginx como proxy inverso.

Todas las respuestas son enviadas en formato JSON.

Son consumidas por el front por el archivo de rutas en config 

---

# 1. Autenticación

## Endpoint

```http id="i2h95d"
POST /api/login
```

---

## Descripción

Valida las credenciales de un usuario y devuelve información básica de autenticación.

---

## Cuerpo de la petición

```json id="wb8dqy"
{
  "email": "operario@logistica.com",
  "password": "1234"
}
```

---

## Respuesta exitosa

```json id="n4y3tu"
{
  "success": true,
  "role": "operario"
}
```

---

## Respuesta inválida

```json id="2kjywx"
{
  "success": false,
  "role": null
}
```

---

# 3. Gestión de Productos

# 4. Búsqueda de Producto

## Endpoint

```http id="m2y5kl"
GET /api/productos/{codigo}
```

---

## Descripción

Obtiene información detallada y ubicación física de un producto mediante su código universal.

---

## Parámetros

| Parámetro | Tipo   | Descripción               |
| --------- | ------ | ------------------------- |
| codigo    | String | Código único del producto |

---

## Respuesta exitosa

```json id="8aq5hb"
{
  "codigo": "A123",
  "nombre": "Producto A",
  "stock": 15,
  "ubicacion": {
    "pasillo": 7,
    "estanteria": 99,
    "seccion": "B",
    lote
  }
}
```

---

# 5. Gestión de Pedidos

## Endpoint

```http id="7s0k3u"
POST /api/pedidos
```

---

## Descripción

Permite crear un nuevo pedido logístico.

---

## Cuerpo de la petición

```json id="h3lj3n"
{
  "sucursal": "Sucursal Sur",
  "productos": [
    {
      "codigo": "A123",
      "cantidad": 2
    }
  ]
}
```

---

## Respuesta exitosa

```json id="r3g8r0"
{
  "success": true,
  "pedidoId": 15
}
```

# Códigos de Respuesta

| Código | Descripción                |
| ------ | -------------------------- |
| 200    | Solicitud exitosa          |
| 400    | Error de validación        |
| 401    | Credenciales inválidas     |
| 404    | Recurso no encontrado      |
| 500    | Error interno del servidor |




# Gestión de Lotes
Obtener todos los lotes
GET /api/lotes
Descripción

Obtiene la lista de lotes registrados en el sistema.

Respuesta
[
{
"idLote":"L001",
"producto":"Coca Cola 500ml",
"cantidad":100
}
]
Consultar un lote
Endpoint
GET /api/lotes/{idLote}
Descripción

Obtiene información detallada de un lote específico.

Respuesta
{
"idLote":"L001",
"producto":"Coca Cola 500ml",
"cantidad":100,
"ubicacionActual":"Pasillo 5"
}
Consultar trazabilidad de un lote
Endpoint
GET /api/lotes/{idLote}/movimientos
Descripción

Obtiene el historial completo de movimientos asociados a un lote.

Respuesta
[
{
"tipo":"INGRESO",
"origen":"Recepción",
"destino":"Depósito Central"
},
{
"tipo":"TRASLADO",
"origen":"Depósito Central",
"destino":"Pasillo 5"
},
{
"tipo":"PREPARACION_PEDIDO",
"origen":"Pasillo 5",
"destino":"Pedido 45"
}
]
Registrar movimiento
Endpoint
POST /api/lotes/{idLote}/movimientos
Descripción

Registra un nuevo movimiento para un lote y lo almacena en el historial y en la pila de operaciones recientes.

Cuerpo
{
"tipo":"TRASLADO",
"origen":"Pasillo 5",
"destino":"Expedición"
}
Revertir último movimiento
Endpoint
POST /api/lotes/{idLote}/undo
Descripción

Revierte el último movimiento registrado para el lote utilizando la pila de movimientos.

Respuesta
{
"success": true,
"mensaje": "Último movimiento revertido correctamente"
}
